# 项目初始化操作手册

> 这份手册已精简。核心操作步骤已整合到 [README.md](README.md) 的"最快上手"章节。
> 以下是补充说明和进阶内容。

## 快速开始（三步完成）

```bash
# 1. 从模板创建（或用 make scaffold 从需求生成）
bash scripts/init-project.sh fastapi-postgres-redis ~/workspace/my-api my-api

# 2. 进入项目、复制配置
cd ~/workspace/my-api
cp .env.example .env
# 编辑 .env，修改密码和配置

# 3. 初始化并启动
make bootstrap && make dev
```

详细说明见 [README.md](README.md)。

## 环境要求

所有脚本基于 Bash，需要以下任一环境：

- **Windows + WSL2 Ubuntu**（推荐）：在 WSL 终端中直接执行
- **原生 Ubuntu/Linux**：直接在终端中执行
- **Windows PowerShell**：通过 `wsl bash scripts/xxx.sh` 调用

> 如果你是全新 Ubuntu 环境，可以先运行 `bash scripts/setup.sh` 一键安装开发工具。

## 命令速查

| 命令 | 用途 |
|------|------|
| `make bootstrap` | 环境检查 + 项目初始化（推荐首次使用） |
| `make check` | 只做环境检查 |
| `make dev` | 启动开发 |
| `make test` | 运行测试 |
| `make validate` | 校验 project-spec.yaml |
| `make scaffold` | 基于 project-spec.yaml 生成项目骨架 |
| `make sync-lib` | 同步 lib.sh 到所有模板 |
| `make init-project` | 显示从模板创建项目的用法 |

## 模板选择

| 模板 | 适合场景 |
|------|----------|
| `fastapi-postgres-redis` | 最小后端 API、AI 后端服务、单人 MVP |
| `fastapi-postgres-redis-alembic` | 需要数据库模型和迁移管理的后端项目 |
| `react-fastapi-postgres-redis` | 前后端分离 Web 项目、管理后台、SaaS 原型 |

更详细的对比见 `doc/01-onboarding/模板案例总览.md`。

## 常见误区

1. **在蓝图根目录直接 `make dev`** → 先从模板创建项目或用 `make scaffold` 生成
2. **复制模板后手动改配置** → 用 `init-project.sh` 一键替换所有占位符
3. **不修改 .env 中的默认密码** → 必须替换为强密码
4. **在 Windows PowerShell 中直接运行脚本** → 通过 WSL 执行
