# 使用总览导航

这份文档是 `_agent_blueprint` 的阅读导航。

如果你第一次打开这个目录，不确定这么多文档到底该先看哪一个，就先看这份。

如果你只是想先理解 `doc/` 目录每一层分别放什么，可以先看 `doc/README.md`。

它不负责定义具体规则，而是负责告诉你：

- 你现在处于什么阶段
- 应该先读哪份文档
- 下一步该把 Agent 引导到哪里
- `templates/` 什么时候才该介入

## 1. 先记住一个原则

- 根目录和 `doc/` 下的规则文档，才是默认主入口
- `templates/` 是可选样例，不是默认上下文
- 没有明确决定使用某个模板前，不要先把模板内容读进来

## 2. 最常见的 5 个入口场景

### 场景 A：第一次使用这套体系

按这个顺序看：

1. `README.md`
2. `项目初始化操作手册.md`
3. `AGENTS.md`

适合：

- 第一次接触这套体系
- 想先搞明白整体工作方式
- 还没开始具体任务

如果你后续还要继续维护这套文档结构，再补看：

4. `doc/04-maintenance/文档命名与目录规范.md`
5. `doc/04-maintenance/术语使用规范.md`
6. `doc/04-maintenance/最小维护说明.md`
7. `doc/04-maintenance/项目P0迭代路线图.md`
8. `doc/01-onboarding/能力边界与支持范围.md`

### 场景 B：准备新开一个项目

按这个顺序看：

1. `项目初始化操作手册.md`
2. `project-spec.example.yaml`
3. `AGENTS.md`
4. `.env.example`
5. `compose.yaml`
6. `scripts/`

适合：

- 从零开始一个新项目
- 想先明确需求、环境和初始化流程

### 场景 C：已经知道要做什么，但想把任务说清楚

按这个顺序看：

1. `doc/02-workflow/任务输入模板.md`
2. `AGENTS.md`
3. `project-spec.yaml`，如果存在

适合：

- 想减少来回沟通
- 希望提前说清范围、限制和完成标准

### 场景 D：要执行一个具体任务

按这个顺序看：

1. `AGENTS.md`
2. `doc/02-workflow/任务执行与交付验收准则.md`
3. `doc/task-playbooks/README.md`
4. 对应任务手册

适合：

- 新增接口
- 改表和迁移
- 接入第三方 API
- 做验证和回归检查

### 场景 E：任务做完了，要稳定汇报结果

按这个顺序看：

1. `doc/02-workflow/任务结果输出模板.md`
2. `doc/02-workflow/任务执行与交付验收准则.md`

适合：

- 想统一最终交付话术
- 想减少“只说完成、不说验证”的情况

## 3. 一张最短阅读地图

如果只想记最短路径，可以按这个理解：

- 想知道这套仓库是干什么的：看 `README.md`
- 想知道怎么起一个新项目：看 `项目初始化操作手册.md`
- 想知道 Agent 的总规则：看 `AGENTS.md`
- 想理解 `doc/` 分层和入口：看 `doc/README.md`
- 想看边界、FAQ 和模板选择：看 `doc/01-onboarding/`
- 想进入任务协作链路：看 `doc/02-workflow/`
- 想查长期资料或维护规范：看 `doc/03-reference/` 和 `doc/04-maintenance/`

## 4. 推荐协作链路

一条最推荐的使用链路是：

1. 先用 `project-spec.yaml` 写清项目需求
2. 再用 `doc/02-workflow/任务输入模板.md` 把当前任务说清楚
3. 让 Agent 先读 `AGENTS.md`
4. 让 Agent 读 `doc/02-workflow/任务执行与交付验收准则.md`
5. 如果任务类型明确，再让 Agent 读 `doc/task-playbooks/` 中的对应手册
6. 最终按 `doc/02-workflow/任务结果输出模板.md` 汇报结果

## 5. 模板什么时候介入

只有下面情况，才建议进入 `templates/`：

- 你已经决定直接从某个成品模板开始
- 你需要一个现成样例来加速搭建
- 根目录规则已经不足以支撑当前初始化任务

不建议进入模板的情况：

- 还在梳理规则
- 还在澄清需求
- 还在设计协作方式
- 只是想补文档、补规范、补执行准则

## 6. 给人和 Agent 的一句话版本

```text
先看总规则，再看当前阶段需要的文档；没有明确选择模板前，不要默认读 templates。
```
