# 能力边界与支持范围

这份文档用于明确说明当前仓库已经支持什么、部分支持什么，以及暂不保证什么。

目标不是把能力说得很大，而是让用户在开始使用前就知道预期。

## 推荐环境

- 推荐：`Windows + WSL2 Ubuntu`
- 支持：原生 `Ubuntu`
- 不推荐：Windows 原生命令环境直接执行 Bash 脚本（需通过 WSL 调用）
- 待补充验证：macOS

## 当前已支持

### 1. 项目启动与协作规则

- `AGENTS.md` 作为 Agent 主规则入口
- `README.md` 和 `项目初始化操作手册.md` 作为人类用户主入口
- `doc/` 下已有导航、输入、执行、交付等辅助文档

### 2. 环境检查

- 通过 `scripts/check-env.sh` 检查本机环境
- 用中文输出已安装、缺失和可选缺失项
- 支持在安装前征求用户确认
- **Docker daemon 运行状态检测**
- **端口冲突检测**（5432/6379 默认端口）

### 3. 项目初始化

- 通过 `scripts/bootstrap.sh` 作为统一主入口
- 初始化时会先检查环境（含 Docker daemon 和端口检测），再执行项目初始化
- 可按项目文件情况自动执行 Python、Node 和 Docker 相关步骤
- Docker daemon 未运行时给出修复建议而非直接报错退出
- 端口冲突时交互确认是否继续

### 4. 模板→项目一键转换

- 通过 `scripts/init-project.sh` 从模板创建新项目
- 自动替换项目名、容器名、数据库名、密码占位符
- 自动初始化 Git 仓库
- 支持 3 个已验证模板

### 5. 需求文件校验

- 通过 `scripts/validate-spec.sh` 校验 `project-spec.yaml`
- 检查必填字段、枚举值、项目名格式、需求概述长度
- 通过 `make validate` 快捷调用

### 6. 本地基础设施

- 通过 `compose.yaml` 管理 Docker 基础服务
- 配置项引用 `.env` 环境变量（带默认值兜底）
- 当前主要覆盖 PostgreSQL 和 Redis
- `COMPOSE_PROJECT_NAME` 用于区分不同项目的 Docker 资源

### 7. 开发与测试入口

- 通过 `scripts/dev.sh` 提供统一开发入口
- 通过 `scripts/test.sh` 提供统一测试入口
- 蓝图根目录下给出清晰引导而非只说"不行"
- 模板已提供更具体的开发与测试脚本

### 8. 可运行模板

- `templates/fastapi-postgres-redis`
- `templates/fastapi-postgres-redis-alembic`
- `templates/react-fastapi-postgres-redis`

### 9. 共享函数库

- `scripts/lib.sh` 统一工具检测、安装、Docker 检测等函数
- 所有脚本和模板通过 `source` 引入，修改只需更新一处

## 当前部分支持

### 1. 自动安装

- 在 `apt` 可用的 Ubuntu / WSL Ubuntu 环境下支持部分依赖自动安装
- 仍依赖用户授权与当前系统权限
- 对非 `apt` 环境暂不提供同等能力

### 2. 模板覆盖面

- 当前重点覆盖 Python 后端与 React + FastAPI 全栈方向
- 其他技术栈仍需要按需求扩展模板或重新生成骨架

### 3. Agent 自动补全

- 可以帮助用户完成技术选型、骨架补全、脚本完善和文档同步
- 但仍需要用户在关键架构选择上拍板

### 4. 数据库迁移

- Alembic 模板已支持迁移工作流
- 其他 ORM 或数据库的迁移待扩展

## 当前暂不保证

- macOS 全流程验证
- Windows 原生命令环境自动安装
- 全技术栈模板覆盖
- 完整 CI/CD
- 自动部署
- 线上监控、日志、告警等运维体系
- 从需求到上线全链路无人值守

## 使用建议

- 如果你知道要用哪个模板：`bash scripts/init-project.sh <模板名> <目录>`
- 如果你是新手、不确定模板：默认先执行 `make bootstrap`
- 如果你只想先看看电脑环境：执行 `make check`
- 如果你需要校验需求文件：执行 `make validate`
- 如果你要向团队介绍这个仓库：先让他们看 `README.md` 和本文件
- 如果你担心失败后不知道怎么办：看 `doc/01-onboarding/常见问题与失败处理.md`

## 安全提醒

- `compose.yaml` 中的密码引用 `.env` 环境变量，不再硬编码
- `.env.example` 中的密码是占位符，必须替换为强密码
- `.env` 已加入 `.gitignore`，不会被提交到仓库
