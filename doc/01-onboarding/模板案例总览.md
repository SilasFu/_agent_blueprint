# 模板案例总览

这份文档用于把当前仓库已经整理好的 3 个模板样例集中展示出来。

如果你是第一次进入仓库，又不想一上来读太多实现细节，可以先看这份文档，快速判断哪个模板更适合你。

## 一眼看懂

| 模板 | 最适合的项目 | 默认主入口 | 运行后能看到什么 |
| --- | --- | --- | --- |
| `fastapi-postgres-redis` | 最小后端 API、AI 后端服务、单人 MVP | `make bootstrap` | FastAPI 服务、`/health`、`/docs` |
| `fastapi-postgres-redis-alembic` | 需要数据库模型和迁移管理的后端项目 | `make bootstrap` | FastAPI 服务、`/users`、`/items`、Alembic 迁移 |
| `react-fastapi-postgres-redis` | 前后端分离 Web 项目、管理后台、SaaS 原型 | `make bootstrap` | React 页面、FastAPI 后端、`/api/message` |

## 选择决策树

```text
你现在要做的是哪类项目？
|
+-- 只做后端 API 或 AI 后端服务
|   |
|   +-- 目前只想快速起步，先跑通最小服务
|   |   -> 选 fastapi-postgres-redis
|   |
|   +-- 一开始就需要数据库模型和迁移管理
|       -> 选 fastapi-postgres-redis-alembic
|
+-- 需要前端页面 + 后端接口一起开发
    |
    +-- 需要最小可运行全栈样例
        -> 选 react-fastapi-postgres-redis
```

## 1. FastAPI 最小后端模板

路径：

- `templates/fastapi-postgres-redis`

适合：

- AI 后端服务
- 中小型 API 项目
- 想快速验证一个 Python 后端起点的人

默认流程：

```bash
make bootstrap
make dev
make test
```

运行成功后：

- 可以访问 `http://127.0.0.1:8000/docs`
- 可以访问 `http://127.0.0.1:8000/health`
- 根路径 `/` 会返回项目名称和环境信息

典型返回：

```json
{
  "message": "FastAPI template is running",
  "app": "app",
  "environment": "dev"
}
```

适合作为：

- “从 0 到最小后端项目”的入门样例

## 2. FastAPI + Alembic 增强后端模板

路径：

- `templates/fastapi-postgres-redis-alembic`

适合：

- 需要数据库模型和迁移管理的项目
- 更接近真实业务后端结构的项目
- 预计会长期演进的数据型项目

默认流程：

```bash
make bootstrap
make dev
make test
make migrate
```

运行成功后：

- 可以访问 `http://127.0.0.1:8000/docs`
- 可以访问 `http://127.0.0.1:8000/users`
- 可以访问 `http://127.0.0.1:8000/items`
- 初始化时会自动执行 Alembic 迁移

典型返回：

```json
{
  "message": "FastAPI Alembic template is running",
  "app": "app",
  "environment": "dev"
}
```

适合作为：

- “从 0 到更真实后端结构”的样例

## 3. React + FastAPI 全栈模板

路径：

- `templates/react-fastapi-postgres-redis`

适合：

- 前后端分离 Web 项目
- 管理后台
- SaaS 原型
- 需要同时维护前端页面和后端 API 的项目

默认流程：

```bash
make bootstrap
make dev-backend
make dev-frontend
make test
```

运行成功后：

- 前端地址：`http://127.0.0.1:5173`
- 后端文档：`http://127.0.0.1:8000/docs`
- 页面默认会显示 `React + FastAPI Template`
- 当后端正常连接时，页面会显示：

```text
Hello from FastAPI backend
```

适合作为：

- “从 0 到最小全栈项目”的样例

## 该怎么选

- 如果你只想先验证一个最小后端服务，选 `fastapi-postgres-redis`
- 如果你一开始就知道自己需要数据库迁移，选 `fastapi-postgres-redis-alembic`
- 如果你需要同时验证前端页面和后端接口，选 `react-fastapi-postgres-redis`

## 下一步建议

- 选好模板后，先阅读对应模板下的 `README.md`
- 如果你还没有最终确定技术栈，先回到根目录 `README.md`
- 如果你想从规则体系开始，而不是直接从模板起步，先看 `项目初始化操作手册.md`
