# 模板案例总览

这份文档用于把当前仓库已经整理好的 3 个模板样例集中展示出来。

如果你是第一次进入仓库，又不想一上来读太多实现细节，可以先看这份文档，快速判断哪个模板更适合你。

## 一眼看懂

| 模板 | 最适合的项目 | 一键创建命令 |
| --- | --- | --- |
| `fastapi-postgres-redis` | 最小后端 API、AI 后端服务、单人 MVP | `bash scripts/init-project.sh fastapi-postgres-redis <目录> <项目名>` |
| `fastapi-postgres-redis-alembic` | 需要数据库模型和迁移管理的后端项目 | `bash scripts/init-project.sh fastapi-postgres-redis-alembic <目录> <项目名>` |
| `react-fastapi-postgres-redis` | 前后端分离 Web 项目、管理后台、SaaS 原型 | `bash scripts/init-project.sh react-fastapi-postgres-redis <目录> <项目名>` |

## 选择决策树

```text
你现在要做的是哪类项目？
|
+-- 只做后端 API 或 AI 后端服务
|   |
|   +-- 目前只想快速起步，先跑通最小服务
|   |   -> 选 fastapi-postgres-redis
|   |
|   +-- 一开始就需要数据库模型和迁移管理
|       -> 选 fastapi-postgres-redis-alembic
|

    |
    +-- 需要最小可运行全栈样例
        -> 选 react-fastapi-postgres-redis
```

## 1. FastAPI 最小后端模板

路径：

- `templates/fastapi-postgres-redis`

适合：

- AI 后端服务
- 中小型 API 项目
- 想快速验证一个 Python 后端起点的人

推荐创建方式：

```bash
bash scripts/init-project.sh fastapi-postgres-redis ~/workspace/my-api my-api
cd ~/workspace/my-api
cp .env.example .env
# 编辑 .env，修改密码和配置
make bootstrap
make dev
make test
```

运行成功后：

- 可以访问 `http://127.0.0.1:8000/docs`
- 可以访问 `http://127.0.0.1:8000/health`
- 根路径 `/` 会返回项目名称和环境信息

## 2. FastAPI + Alembic 增强后端模板

路径：

- `templates/fastapi-postgres-redis-alembic`

适合：

- 需要数据库模型和迁移管理的项目
- 更接近真实业务后端结构的项目
- 预计会长期演进的数据型项目

推荐创建方式：

```bash
bash scripts/init-project.sh fastapi-postgres-redis-alembic ~/workspace/my-api my-api
cd ~/workspace/my-api
cp .env.example .env
# 编辑 .env，修改密码和配置
make bootstrap
make dev
make test
bash scripts/migrate.sh
```

运行成功后：

- 可以访问 `http://127.0.0.1:8000/docs`
- 可以访问 `http://127.0.0.1:8000/users`
- 可以访问 `http://127.0.0.1:8000/items`

## 3. React + FastAPI 全栈模板

路径：

- `templates/react-fastapi-postgres-redis`

适合：

- 前后端分离 Web 项目
- 管理后台
- SaaS 原型
- 需要同时维护前端页面和后端 API 的项目

推荐创建方式：

```bash
bash scripts/init-project.sh react-fastapi-postgres-redis ~/workspace/my-app my-app
cd ~/workspace/my-app
cp .env.example .env
# 编辑 .env，修改密码和配置
make bootstrap
make dev
make test
```

运行成功后：

- 前端地址：`http://127.0.0.1:5173`
- 后端文档：`http://127.0.0.1:8000/docs`

## 该怎么选

- 如果你只想先验证一个最小后端服务，选 `fastapi-postgres-redis`
- 如果你一开始就知道自己需要数据库迁移，选 `fastapi-postgres-redis-alembic`
- 如果你需要同时验证前端页面和后端接口，选 `react-fastapi-postgres-redis`

## 下一步建议

- 选好模板后，使用 `init-project.sh` 一键创建项目
- 创建完成后，先编辑 `.env` 修改密码和配置
- 然后执行 `make bootstrap` 初始化
- 如果你想从规则体系开始，而不是直接从模板起步，先看 `项目初始化操作手册.md`
