# 常见问题与失败处理

这份文档用于处理当前仓库最常见的初始化失败场景。

如果你遇到问题，先不要急着怀疑整个项目不可用，先看是不是属于下面这些高频情况。

## 快速 FAQ

- 第一次进入仓库先运行什么：
  - 默认先运行 `make bootstrap`
- 什么时候才需要 `make check`：
  - 当你只想单独体检环境时
- 不知道该选哪个模板怎么办：
  - 先看 `doc/01-onboarding/模板案例总览.md`
- 遇到自动安装失败怎么办：
  - 先确认是否在 `Windows + WSL2 Ubuntu` 或原生 `Ubuntu` 环境中
- 初始化成功后下一步做什么：
  - 优先执行 `make dev`、`make test`，或按模板提示执行前后端启动命令

## 1. 运行脚本后提示没有 `apt`

典型表现：

- 提示当前系统没有检测到 `apt`
- 提示当前环境暂不支持自动安装

原因：

- 你当前不在 Ubuntu 或 WSL Ubuntu 环境中

怎么处理：

- 优先切换到 `Windows + WSL2 Ubuntu`
- 或者切换到原生 `Ubuntu`
- 如果你必须停留在当前系统，请改为手动安装缺失工具

是否还能继续：

- 可以继续阅读文档
- 但自动安装能力通常不能继续使用

## 2. 运行脚本后提示没有 `sudo`

典型表现：

- 安装时提示没有权限
- 需要提升权限才能安装依赖

原因：

- 当前用户没有管理员权限
- 当前环境不允许使用 `sudo`

怎么处理：

- 使用具备管理员权限的用户
- 或者让管理员先安装 Git、Docker、Python、Node 等基础依赖

是否还能继续：

- 只要缺的是必需工具，通常不能继续初始化

## 3. Docker 已安装，但命令仍然失败

典型表现：

- `docker` 已安装
- 但运行 `docker compose up -d` 时提示权限不足

原因：

- Docker 用户组变更还未在当前登录会话生效

怎么处理：

- 重新打开终端
- 重新登录 WSL 或系统会话
- 必要时先使用 `sudo docker ...` 验证 Docker 是否可用

是否还能继续：

- 在当前终端里可能不稳定
- 建议刷新会话后再继续

## 4. `nvm` 或 `pyenv` 安装后提示命令还不可用

典型表现：

- 脚本显示安装完成
- 但当前终端执行命令仍提示未找到

原因：

- 当前 shell 还没有重新加载配置文件

怎么处理：

- 重新打开终端
- 或者执行 `source ~/.bashrc`

是否还能继续：

- 部分情况下不能继续
- 建议先让当前 shell 重新加载环境后再继续

## 5. `uv` 安装后仍然找不到命令

典型表现：

- 安装脚本执行成功
- 但继续执行时找不到 `uv`

原因：

- `~/.local/bin` 还没有进入当前 PATH

怎么处理：

- 重新打开终端
- 或者执行 `source ~/.bashrc`
- 再次执行 `uv --version` 确认是否可用

是否还能继续：

- 如果当前项目依赖 Python 初始化，通常不能继续

## 6. `docker compose up -d` 失败

典型表现：

- Docker 已安装
- 但容器拉取、启动或端口绑定失败

常见原因：

- 网络无法拉取镜像
- 端口已被占用
- Docker 服务没有正常启动
- compose 文件中的环境变量尚未补齐

怎么处理：

- 先确认 Docker 服务可用
- 再检查端口是否冲突
- 再检查 `.env` 是否已经创建并补齐关键配置
- 必要时单独运行 `docker compose up` 观察详细日志

是否还能继续：

- 项目骨架可以继续查看
- 但依赖数据库或缓存时，开发体验会受影响

## 7. 不知道应该先运行 `check` 还是 `bootstrap`

默认规则：

- 如果你是第一次使用，优先运行 `make bootstrap`
- 如果你只想单独做环境体检，再运行 `make check`

原因：

- `bootstrap` 是默认主入口
- `check` 是辅助入口

## 8. 模板已经初始化成功，但不知道下一步做什么

默认顺序：

- 运行 `make dev`
- 或根据模板提示分别运行 `make dev-backend`、`make dev-frontend`
- 然后运行 `make test`

如果你仍然不确定：

- 先阅读对应模板下的 `README.md`
- 再让 Agent 根据当前模板补齐更详细的开发和测试入口说明
