# 新电脑开发环境一页式速查清单

适用目标：

- 新电脑到手后，快速搭建 `Windows + WSL2 + Ubuntu`
- 在 `WSL Ubuntu` 里装好 `Python + Node.js + pnpm`
- 尽量不污染 Windows 原系统

---

## 1. 总原则

- `Windows`：只装桌面软件、IDE、终端
- `Ubuntu`：装开发工具链
- `Docker`：按需装，按需开
- 项目统一放在 `~/workspace`

---

## 2. Windows 侧安装

先在管理员 `PowerShell` 执行：

```powershell
wsl --install -d Ubuntu
```

重启电脑后检查：

```powershell
wsl --status
wsl -l -v
```

正常结果类似：

```text
NAME      STATE           VERSION
Ubuntu    Stopped         2
```

如果开始菜单搜不到 `Ubuntu`，直接执行：

```powershell
wsl -d Ubuntu
```

---

## 3. 第一次进入 Ubuntu

第一次启动时会提示：

```text
Create a default Unix user account:
```

依次完成：

1. 输入 Linux 用户名
2. 输入 Linux 密码
3. 再输入一次确认密码

注意：

- 输密码时屏幕不显示字符，正常

---

## 4. Ubuntu 初始化

进入 Ubuntu 后执行：

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y build-essential curl wget git unzip zip ca-certificates pkg-config software-properties-common
mkdir -p ~/workspace
cd ~/workspace
pwd
```

正常情况下：

```bash
/home/你的用户名/workspace
```

---

## 5. 安装 Python

先装依赖：

```bash
sudo apt install -y make build-essential libssl-dev zlib1g-dev libbz2-dev libreadline-dev libsqlite3-dev curl llvm libncursesw5-dev xz-utils tk-dev libxml2-dev libxmlsec1-dev libffi-dev liblzma-dev
```

安装 `pyenv`：

```bash
curl https://pyenv.run | bash
```

写入 `~/.bashrc`：

```bash
cat >> ~/.bashrc <<'EOF'
export PYENV_ROOT="$HOME/.pyenv"
[[ -d $PYENV_ROOT/bin ]] && export PATH="$PYENV_ROOT/bin:$PATH"
eval "$(pyenv init - bash)"
EOF
```

生效并安装 Python：

```bash
source ~/.bashrc
pyenv install 3.12.9
pyenv global 3.12.9
pyenv rehash
python --version
```

安装 `uv`：

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
source ~/.bashrc
source $HOME/.local/bin/env
uv --version
```

---

## 6. 安装 Node.js

安装 `nvm`：

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
source ~/.bashrc
nvm --version
```

安装 Node LTS：

```bash
nvm install --lts
nvm use --lts
node -v
npm -v
```

安装 `pnpm`：

```bash
corepack enable
corepack prepare pnpm@latest --activate
pnpm -v
```

---

## 7. VS Code 推荐方式

Windows 里安装：

- `VS Code`
- 扩展 `Remote - WSL`

以后在 Ubuntu 项目目录中执行：

```bash
code .
```

## 8. WSL 资源限制

在 Windows `PowerShell` 执行：

```powershell
notepad $env:USERPROFILE\.wslconfig
```

填入：

```ini
[wsl2]
memory=4GB
processors=3
swap=1GB
localhostForwarding=true
```

保存后执行：

```powershell
wsl --shutdown
```

通过标准：

- `python` 正常

- `pnpm` 正常
- 当前目录是 `~/workspace`

## 10. 高频坑

### 1. 看不见 Ubuntu

用：

```powershell
wsl -d Ubuntu
```

### 2. `wsl -l -v` 没有分发

说明只装了 WSL 平台，还没装 Ubuntu：

```powershell
wsl --install -d Ubuntu
```

### 3. 密码输入没显示

正常，直接输入并回车。

### 4. 把密码当成命令输入

错误现象：

```bash
123456: command not found
```

说明你不是在密码提示状态。

### 5. URL 误加反引号

错误：

```bash
curl -o- `https://...` | bash
```

正确：

```bash
curl -o- https://... | bash
```

### 6. `python` 找不到

执行：

```bash
pyenv global 3.12.9
pyenv rehash
source ~/.bashrc
```

### 7. `uv` 找不到

执行：

```bash
source ~/.bashrc
source $HOME/.local/bin/env
```

### 8. `nvm` 找不到

执行：

```bash
source ~/.bashrc
```
