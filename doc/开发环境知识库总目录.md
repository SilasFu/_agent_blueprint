# 开发环境知识库总目录

这份文档是 `_agent_blueprint/doc` 下长期资料的分组索引页。

它保留“开发环境知识库总目录”这个名称，是为了兼容已有入口；但现在它承担的是更广义的知识库导航作用。

适用目标：

- 快速找到你需要的那份手册
- 按正确顺序完成新电脑环境配置
- 在做项目、配 Git、做备份时少走弯路
- 了解 Agent 协作规则类文档和环境知识库资料的分工

---

## 4. 一套标准工作流

建议你长期遵循这条工作流：

1. 新电脑到手后，先按 `一页式速查` 装好 `WSL Ubuntu`
2. 再按 `完整教材` 装好 `Python`、`Node`、`pnpm`
3. 按 `SSH 教程` 配好 `GitHub` 或 `Gitee`
4. 新项目开始前，先看 `使用总览导航` 和 `AGENTS.md`
5. 用 `任务输入模板` 把当前任务说清楚
6. 再按 `Python 模板` 或其他项目资料起步
7. 需要数据库和缓存时，套用 `Docker + PostgreSQL + Redis` 模板
8. 日常开发时按 `Git 远程同步手册` 做 `commit`、`push`、备份

## 5. 知识库分组原则

- 规则类文档优先放在 `doc/` 根层
- 高频具体任务统一放到 `doc/task-playbooks/`
- 环境、Git、备份、项目模板等长期资料继续放在 `doc/` 根层
- 成品代码样例继续放在 `templates/`，不要和规则文档混在一起

---

## 6. 最重要的几条原则

- `Windows 做宿主机`
- `Ubuntu 做开发环境`
- `项目放在 ~/workspace`
- `代码必须 push 到远程仓库`
- `.env` 不进 Git，但 `.env.example` 要保留
- 数据库数据不能只指望 Docker 卷
- `WSL` 可以重建，不要把它当唯一备份
