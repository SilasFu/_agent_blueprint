# doc 目录索引

这份文件是 `doc/` 目录本身的入口说明。

如果你点进 `doc/` 后一眼看到很多子目录，不确定该先看哪里，就先看这份索引。

## 这个目录怎么理解

当前 `doc/` 已按用途分成 5 层：

- `01-onboarding`：第一次进入仓库时最该先看的导航、边界、FAQ、案例
- `02-workflow`：任务怎么提、怎么执行、怎么汇报
- `03-reference`：环境、Git、备份、模板等长期参考资料
- `04-maintenance`：命名规范、术语规范、维护说明、路线图
- `task-playbooks`：新增接口、迁移、第三方接入、回归验证等高频任务手册

## 你该先看哪一层

| 你的目标 | 先看哪里 |
| --- | --- |
| 第一次打开仓库，不知道顺序 | `01-onboarding/使用总览导航.md` |
| 想起一个新项目 | 根目录 `项目初始化操作手册.md` |
| 想把任务描述清楚 | `02-workflow/任务输入模板.md` |
| 想规范执行过程 | `02-workflow/任务执行与交付验收准则.md` |
| 想统一结果汇报 | `02-workflow/任务结果输出模板.md` |
| 想快速比较模板 | `01-onboarding/模板案例总览.md` |
| 想查环境或 Git 资料 | `03-reference/开发环境知识库总目录.md` |
| 想维护这套体系本身 | `04-maintenance/文档命名与目录规范.md` |
| 想执行具体高频任务 | `task-playbooks/README.md` |

## 推荐阅读路径

### 场景 1：第一次进入这个仓库

1. 先看 `01-onboarding/使用总览导航.md`
2. 再看根目录 `README.md`
3. 然后看根目录 `项目初始化操作手册.md`
4. 最后让 Agent 读取根目录 `AGENTS.md`

### 场景 2：准备起一个新项目

1. 先看根目录 `项目初始化操作手册.md`
2. 再看 `01-onboarding/能力边界与支持范围.md`
3. 如果还没决定模板，再看 `01-onboarding/模板案例总览.md`
4. 最后执行 `make bootstrap`

### 场景 3：准备让 Agent 执行具体任务

1. 先看 `02-workflow/任务输入模板.md`
2. 再看 `02-workflow/任务执行与交付验收准则.md`
3. 如果任务类型明确，再看 `task-playbooks/README.md`
4. 完成后按 `02-workflow/任务结果输出模板.md` 汇报

## 每一层里放什么

### `01-onboarding`

- 适合第一次接触这套体系的人
- 解决“先看什么”“支持到哪一步”“出了问题怎么办”“模板怎么选”

### `02-workflow`

- 适合已经准备开始协作的人
- 解决“任务怎么提”“执行过程怎么控质量”“结果怎么汇报”

### `03-reference`

- 适合需要查长期资料的人
- 解决环境搭建、Git、SSH、备份、模板背景资料等问题

### `04-maintenance`

- 适合维护这套仓库本身的人
- 解决命名、分层、术语、路线图和最小维护规则

### `task-playbooks`

- 适合已经明确任务类型的人
- 解决新增接口、改表迁移、第三方接入、改动后验证等高频任务

## 一句话记忆

- 第一次来：先看 `01-onboarding`
- 要协作：看 `02-workflow`
- 要查资料：看 `03-reference`
- 要维护体系：看 `04-maintenance`
- 要做具体任务：看 `task-playbooks`
