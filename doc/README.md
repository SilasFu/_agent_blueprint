# doc 目录索引

这份文件只负责一件事：

- 帮你快速理解 `doc/` 下面每一层分别放什么

如果你已经知道自己要做什么，就直接进入对应层级；如果你还不知道阅读顺序，再去看 `01-onboarding/使用总览导航.md`。

## 先区分 3 个词

- `索引`：解释一个目录里都有什么，适合做目录入口
- `导航`：告诉你先看什么、再看什么，适合做阅读分流
- `总目录`：汇总某一类长期资料，适合做知识库入口

当前对应关系是：

- `doc/README.md`：`doc/` 目录索引
- `01-onboarding/使用总览导航.md`：阅读导航
- `03-reference/开发环境知识库总目录.md`：长期资料总目录

## 这个目录怎么理解

当前 `doc/` 已按用途分成 5 层：

| 层级 | 主要用途 | 典型入口 |
| --- | --- | --- |
| `01-onboarding` | 第一次进入仓库时的阅读入口、边界、FAQ、案例 | `使用总览导航.md` |
| `02-workflow` | 任务输入、执行准则、结果输出 | `任务输入模板.md` |
| `03-reference` | 环境、Git、备份、模板等长期参考资料 | `开发环境知识库总目录.md` |
| `04-maintenance` | 命名、术语、维护说明、路线图 | `文档命名与目录规范.md` |
| `task-playbooks` | 明确任务类型后的高频操作手册 | `README.md` |

## 你该先进哪一层

| 你的目标 | 先看哪里 |
| --- | --- |
| 第一次打开仓库，不知道顺序 | `01-onboarding/使用总览导航.md` |
| 想起一个新项目 | 根目录 `项目初始化操作手册.md` |
| 想把任务描述清楚 | `02-workflow/任务输入模板.md` |
| 想规范执行过程 | `02-workflow/任务执行与交付验收准则.md` |
| 想统一结果汇报 | `02-workflow/任务结果输出模板.md` |
| 想快速比较模板 | `01-onboarding/模板案例总览.md` |
| 想查环境或 Git 资料 | `03-reference/开发环境知识库总目录.md` |
| 想维护这套体系本身 | `04-maintenance/文档命名与目录规范.md` |
| 想执行具体高频任务 | `task-playbooks/README.md` |

## 一句话记忆

- 看目录分层：用这个索引
- 看阅读顺序：去 `01-onboarding`
- 查长期资料：去 `03-reference`
- 做具体任务：去 `task-playbooks`
