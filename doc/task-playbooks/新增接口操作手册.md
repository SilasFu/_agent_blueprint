# 新增接口操作手册

这份手册用于约束 Agent 在“新增接口”任务中的标准动作，避免只写控制器或只补一半逻辑。

## 1. 适用场景

- 新增后端 API
- 新增供前端调用的业务接口
- 新增内部服务调用入口

## 2. 开始前先确认

先判断下面几点：

- 这个接口解决什么业务问题
- 调用方是谁
- 输入输出结构是什么
- 是否需要鉴权
- 是否会影响数据库、缓存、第三方 API
- 是否需要补文档和测试

如果接口职责不清、路径设计冲突、鉴权方案不明确，应先提问。

## 3. 标准执行顺序

1. 阅读相关规则和需求文档
2. 确认接口目标、输入、输出、错误场景
3. 检查是否已有相近接口可复用
4. 设计路由、参数、响应结构
5. 补业务层或服务层逻辑
6. 如有需要，补数据库访问或缓存处理
7. 补最小测试或最小验证方式
8. 同步更新 README、接口说明或调用示例

## 4. 至少应同步检查的内容

- 路由文件或入口文件
- 请求参数校验
- 响应模型或返回结构
- 异常处理
- 权限或鉴权依赖
- 日志或关键错误信息
- 测试入口或验证方式

## 5. 常见遗漏

- 只加路由，不补 schema 或参数校验
- 只返回成功结果，不处理异常分支
- 接口加完了，但 README 和调用方式没有更新
- 改了响应结构，却没有同步前端或测试

## 6. 最低验收标准

- 能明确说明接口用途
- 输入输出结构清晰
- 关键异常场景有处理
- 调用方式或验证方式已写明
- 相关文档已同步

## 7. 推荐提示词

```text
先阅读 AGENTS.md、project-spec.yaml、doc/任务执行与交付验收准则.md，
再按 doc/task-playbooks/新增接口操作手册.md 的顺序完成任务。
新增接口时请同时处理参数校验、响应结构、异常处理、必要测试和文档更新。
如果路径设计、鉴权或返回结构存在关键分歧，请先向我确认。
```
