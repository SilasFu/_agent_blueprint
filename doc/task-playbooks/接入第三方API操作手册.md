# 接入第三方API操作手册

这份手册用于约束 Agent 在接入外部服务时的标准动作，避免只写调用代码、不处理密钥、超时、重试和失败场景。

## 1. 适用场景

- 接入大模型服务
- 接入支付、短信、邮件、地图、存储等第三方服务
- 接入外部 REST API、SDK 或 webhook

## 2. 开始前先确认

先判断下面几点：

- 第三方服务解决什么问题
- 用 HTTP API 还是官方 SDK
- 需要哪些环境变量或密钥
- 是否有超时、限流、重试、幂等和计费风险
- 失败后系统应如何降级

如果供应商选择、成本、安全要求或接入方式存在明显分歧，应先提问。

## 3. 标准执行顺序

1. 阅读规则、需求和现有集成方式
2. 明确配置项、密钥、端点和调用边界
3. 设计统一的接入层，不要把调用散落到业务代码里
4. 补环境变量说明和 `.env.example`
5. 实现超时、错误处理和必要重试
6. 为调用结果设计日志、错误提示或降级策略
7. 更新 README 或接入说明
8. 做最小验证，确认配置缺失和失败场景有可理解反馈

## 4. 至少应同步检查的内容

- 配置文件和环境变量模板
- 统一客户端或适配层
- 错误处理和超时控制
- 日志与可观测性
- 文档中的配置说明
- 最小验证脚本或测试

## 5. 常见遗漏

- 把密钥写死在代码里
- 直接在业务逻辑里散落第三方调用
- 只处理成功路径，不处理失败和超时
- 没有在 `.env.example` 中补配置项
- 没说明如何本地验证或替换供应商

## 6. 最低验收标准

- 所需配置已明确
- 第三方调用边界清晰
- 失败路径有处理
- 文档说明了如何配置和验证
- 不把真实密钥提交到仓库

## 7. 推荐提示词

```text
先阅读 AGENTS.md、project-spec.yaml、doc/任务执行与交付验收准则.md，
再按 doc/task-playbooks/接入第三方API操作手册.md 的顺序完成任务。
请把第三方调用封装到统一接入层，并补齐 .env.example、错误处理、超时控制和验证说明。
如果供应商选择、成本、安全或失败降级策略存在关键分歧，请先向我确认。
```
