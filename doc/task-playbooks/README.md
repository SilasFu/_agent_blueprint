# 常见任务操作手册

这组文档不是项目需求说明，而是执行层手册。

它解决的问题是：

- 当 Agent 接到一个具体开发任务时，应该按什么顺序推进
- 哪些检查不能省
- 哪些文件通常需要同步更新
- 最后如何确认任务真的完成

## 使用原则

- 先读 `AGENTS.md`
- 如果任务还没说清楚，先读 `doc/02-workflow/任务输入模板.md`
- 再读 `project-spec.yaml`
- 再读 `doc/02-workflow/任务执行与交付验收准则.md`
- 任务做完后，再参考 `doc/02-workflow/任务结果输出模板.md`
- 最后按当前任务类型选择对应手册

## 当前已提供的手册

- `新增接口操作手册.md`
  - 适用于新增后端 API、页面交互接口、内部服务接口
- `新增数据表与迁移操作手册.md`
  - 适用于新增表结构、字段、索引和迁移脚本
- `接入第三方API操作手册.md`
  - 适用于外部模型、支付、短信、邮件、地图等第三方服务接入
- `改动后验证与回归检查手册.md`
  - 适用于任务完成前的验证、回归、自查和交付说明

## 使用方式

如果当前任务属于明确类型，可以直接这样要求 Agent：

```text
先阅读 AGENTS.md、project-spec.yaml、doc/02-workflow/任务执行与交付验收准则.md，
再阅读 doc/task-playbooks/ 中与当前任务最相关的操作手册。
按手册规定的顺序完成任务，并说明你做了哪些验证。
```

## 一句话记忆

- 先看规则
- 不清楚先补任务输入
- 再选手册
- 再执行任务
- 最后按统一结构交付
